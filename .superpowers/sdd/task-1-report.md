### Deviations from Brief

1. **Tailwind v3 pinned instead of installed-by-default v4.**
   `create-next-app@latest` installed Tailwind v4 (`@tailwindcss/postcss` plugin, no `tailwind.config.ts`, CSS-only config). The brief mandates a verbatim `tailwind.config.ts` with CLF tokens — that API only exists in v3. Downgraded to `tailwindcss@^3.4.19` with `postcss` + `autoprefixer`.

2. **next-intl pinned to `^3` instead of latest (`^4`).**
   next-intl v4 changed `getRequestConfig` to use `requestLocale` (async) and moved the plugin. The brief's snippets match the v3 API. Pinned to `^3.26.5`.

3. **`i18n.ts` → `i18n/request.ts` (with custom path in `createNextIntlPlugin`).**
   next-intl v3.22+ deprecated `./i18n.ts` as the config location in favour of `./i18n/request.ts`. Using the deprecated path only produces a build warning; moving it silences it. Configured `createNextIntlPlugin('./i18n/request.ts')` in `next.config.ts`.

4. **`middleware.ts` → `proxy.ts`.**
   Next.js 16 renamed the file-convention from `middleware` to `proxy` (the old name prints a deprecation warning at build time). Renamed to `proxy.ts`; config export and next-intl import are unchanged.

5. **No GitHub repo created.**
   Instruction override in the task spec says "Do NOT create a GitHub repo yet (no gh CLI call) — just scaffold locally." Step 8 (`gh repo create`) was skipped.

### Known Issues / Warnings

- **`experimental.turbo` warning** — next-intl v3's plugin injects `experimental.turbo` into the Next.js config; Next.js 16 no longer recognises this key and prints a warning. It has no effect on the build and will be resolved when next-intl v4 support is added in a later task.
- **`better-sqlite3` native build skipped** — pnpm 10's new `onlyBuiltDependencies` approval gate blocked the native compilation. `better-sqlite3` ships prebuilt binaries; the native module will build on first `pnpm install` after approving builds or via `pnpm rebuild better-sqlite3`. This does not affect `pnpm build` since no code imports it at compile time.
- **next-intl peer-dep warning** — next-intl 3.26.5 lists `next@"^10…^15"` as a peer; we run Next.js 16. Build still passes; the warning is cosmetic.
